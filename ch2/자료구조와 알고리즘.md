# 자료구조 & 알고리즘

## 자료구조?
- 대량의 데이터를 효율적으로 관리할 수 있는 데이터의 구조
- 코드 상에서 효율적으로 데이터를 처리하기 위해 데이터를 특별한 구조로 만든 것을 의미함
- IT분야에서 현명한 개발자들이 만들어놓은 대표적인 자료구조를 배우면서, 데이터를 구조화하는 방법을 익혀보자.
- 다른 말로는 데이터 구조, Data Structure라고도 부른다.

## 대표적인 자료구조?
- 대표적으로는 다음이 있다.
    - 배열
    - 스택
    - 큐
    - 링크드 리스트
    - 해쉬 테이블
    - 힙
- 현실 세계에서도 대표적인 자료구조가 있다.
    - 사전
    - 우편번호 (5자리)

## 알고리즘?
- 어떤 문제를 풀기 위한 절차 혹은 방법
- 어떤 문제에 대해 특정한 '입력'을 넣으면, 원하는 '출력'을 얻을 수 있도록 만드는 것
- 현실 세계에서도 대표적인 알고리즘이 있다.
    - 요리 레시피
        - 이것도 어떻게 보면 절차이기 때문에 알고리즘의 종류에 속한다.

## 이 두 가지가 왜 중요한가?
- 우리가 프로그래밍을 작성하는 데 있어서 어떤 자료구조와 알고리즘을 쓰느냐에 따라 성능이 천지차이가 나기 때문
- 해당 과정이 프로그래밍을 잘 할 수 있는 기능과 역량을 익힐 수 있고, 검증할 수 있다.
- 회사에서도 이 사람이 개발을 잘하는 지 확인하는 방법으로 쓰인다.
