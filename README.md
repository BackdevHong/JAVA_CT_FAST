## 진행 방식

**해당 부분은 체크리스트를 다 완수하고 난 이후에 진행할 예정입니다.**

1. 강의 내용을 수강하며 핵심 개념을 정리합니다.
2. 예제 코드와 풀이 코드를 직접 작성합니다.
3. 주 1회 코딩테스트 문제를 풉니다.
4. 풀이한 문제는 `Problum-문제번호.md` 형식의 문서로 작성합니다.
5. 작성한 풀이 문서를 기준으로 접근 방식, 시간 복잡도, 개선점을 리뷰합니다.

## 문제 풀이 문서 규칙

문제 풀이 파일은 아래 형식으로 작성합니다.

```text
Problum-문제번호.md
```

예시:

```text
Problum-1000.md
Problum-2557.md
```

각 문서에는 가능한 한 다음 내용을 포함합니다.

- 문제 링크
- 문제 요약
- 접근 방식
- Java 풀이 코드
- 시간 복잡도와 공간 복잡도
- 리뷰 및 개선점

## 목표

- Java 기반 코딩테스트 풀이 감각 회복

- 매주 꾸준히 문제를 풀고 리뷰하는 학습 루틴 만들기
